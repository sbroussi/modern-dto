Presentation of the SOA Catalog

The menu on the left list all the requests and the services (the portal services).

Use Requests by screen to know which request is called using a given screen.

Requests by field is usefull to make a full text search (Ctrl + F) with keywords.

Responses by request is efficient to make impact analysis.

Request by scope classify request within two categories PORTAIL or WEB.

Services by domain and business sort services by functionnal use.

Services by description is usefull to make a full text search (Ctrl + F) with keywords.

Request name format :
  • 2 characters for the department : (RE retail, MP moyen de paiements, ...)
  • S to specify it is a service
  • 3 chars specific to the service
  • 1 char for the action type (C create/Update, V view/read/search, D delete)
  • 0 ...
Example : RES CAS V0 RETAIL, S, Détail d'un compartiment d'un produit d'assurance ( CAS ), Visualisation, 0. SOA Catalog
